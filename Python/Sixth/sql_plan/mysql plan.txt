1：认识mysql
      从服务器讲起，
      为什么我们在微信或者 其他任何软件上能 看到很久以前的消息
      这都是因为部署在云端服务器 中的数据库响应了 客户端的请求
     那么服务器，数据库，客户端是什么？
     简要阐述：
	什么是服务器：
		其实抽象的说我们可以将服务器理解成一个载体，它承载着我们 所编写好的代码，
		以一个website为例：
			website分为前端和后端，
			前端用于 显示信息，目前我们所看见的任何一个网站， 其实都是看见的web site的前端页面
			那么有同学就想问了，那我们还要学习 后端干嘛？，其实后端主要是 做 一个web site的页面信息
			交互，也就是说，如果没有后端 一个web site是会没有任何反应的，就是一个简单的静态页面
			
			那么说到后端；这个可能会比较陌生，应为 我们在接触一个网页的时候 可能从来没有和后端的数据
			打过交道，那么后端 他到底是个什么玩意儿呢？
			后端：
				其实后端他主要分为基本代码，和服务器，
				基本代码就是一个业务流程功能的实现代码，那么服务器是什么呢？以天气网站为例
				我们是不是要通过 一个叫做http:127.0.0.1:5000/的地址访问到 我们所放开的服务
				这就是 后端 flask框架自带的一个 服务器，那么我们 当时 是通过 地址请求到的 
				json数据 json数据是从哪儿来的呢，json数据其实就是从数据库中来的;
		
	什么是数据库
		简单的说就是存放数据的地方，那么为什么不将数据放在文档或者excel中呢，这是应为数据库相对excel有着更多的优势
		常见且主流的数据库有，oracle，mysql，sqlserver，等等，很多很多，不管是什么数据，他们都是采用的
		标准的sql语句，你可以将sql语句理解成另外的一种编程语言，这种编程语言是专门用作用于数据库数据操作的
		虽然有很多种的数据库，但是他们的sql语句相似度都差不多达到95%以上，所以不管你学习哪一种数据库，
		对于你后续的学习，没有太大的影响；
		那么本套课程采用的是mysql数据库，
		mysql，有着自己的优势：免费开源，性能优秀，是一种中大型数据库，
	什么是客户端
		举个例子，当时我们在学习python的时候，是不是分为 pycahrm 和 python 其实他们两的关系就是 代码编辑器和 代码解释器
		那么客户端其实就是连接 mysql 类似于一个编辑器的这么一个玩意儿，在编写sql语句时 会有相应的提示 ，那么在本章 节中会 
		使用到的编辑器是navicat ，来操作数据库
